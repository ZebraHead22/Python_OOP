## 🎓 Система оценивания

Итоговая оценка по курсу рассчитывается по **накопительной системе баллов**. Максимум за курс можно набрать **100%** (при расчёте итоговой оценки проценты выше 100 округляются до 100).

Студент выбирает **один из двух путей** до **первой контрольной работы**. Переключение между путями возможно только по согласованию с преподавателем и **не позднее чем за две недели до конца курса**.

### 1️⃣ Путь через контрольные работы (СИД)

В течение семестра проводятся **две контрольные работы (СИД)**. Они имеют разный вес.

| Работа | Максимум баллов | Вес в курсе |
|--------|-----------------|-------------|
| СИД 1  | 2               | 40%         |
| СИД 2  | 3               | 60%         |

**Максимум за СИДы: 5 баллов** (соответствует **80% итоговой оценки**).

#### Структура СИД

**СИД 1** — две задачи:
- Задача 1: 1 балл
- Задача 2: 1 балл
- Итого: **2 балла**

**СИД 2**:
- Контрольная работа (2 задачи): 2 балла
- Посещение занятий: 1 балл (автоматически при регулярном посещении; допускается **один пропуск без потери балла**)
- Итого: **3 балла**

#### Дополнительные баллы

После СИДов студент может получить оставшиеся **20%** тремя способами:

| Способ               | Баллы | % от курса |
|----------------------|-------|------------|
| Экзамен | 0–2 | 0–20%      |
| Задача повышенной сложности | 0–2 | 0–20%      |
| Творческое задание (мини-проект) | 0–2 | 0–20% |

**Формула расчёта итогового процента** (для пути через контрольные работы):

$$
\% = 0.16 \times (\text{СИД1} + \text{СИД2}) + 0.1 \times (\text{Задачи} + \text{Экзамен})
$$

где **Задачи** и **Экзамен** — баллы за дополнительные испытания (от 0 до 2 каждое).

> 💡 *Примечание:* При максимальных баллах за СИДы (5) для достижения 100% достаточно набрать 2 дополнительных балла. Если СИДы сданы не полностью, потребуется больше дополнительных баллов (см. таблицу примеров).

##### Задача повышенной сложности
Дополнительная задача, выходящая за рамки обычных контрольных. Оценивается по критериям:
- 0 — решение отсутствует или полностью некорректно;
- 1 — задача решена частично, слабая архитектура;
- 2 — полное решение, аккуратный код, хорошая архитектура, проверки входных данных, использование возможностей Python.

##### Творческое задание (мини-проект)
Небольшое приложение, демонстрирующее использование ООП. **Требования:**
- решает конкретную пользовательскую задачу;
- запускается из консоли (`python main.py`);
- имеет простой интерфейс (меню, текстовый UI или минимальный GUI);
- минимум **3 класса**, инкапсуляция, наследование;
- приветствуются `dataclass`, `Enum`, `@property`, пользовательские исключения.

Оценка:
- 0 — программа не работает;
- 1 — программа работает, но архитектура простая;
- 2 — продуманная архитектура, чистый код, хорошая реализация.

#### Экзамен
- Устная форма: теоретические вопросы, небольшая задача, объяснение архитектуры.
- Оценка:
  - 0 баллов — слабые ответы;
  - 1 балл — уверенное знание основных тем;
  - 2 балла — глубокое понимание и корректное решение задачи.

- **ООП-дизайн**:

  - Читаемый, соответствует PEP 8.
  - Присутствуют комментарии и docstring для классов и методов.
  - Обрабатываются ошибочные ситуации (пользовательские исключения, проверки ввода).
- **Сдача проекта**:
  - Исходный код (например, в репозитории GitHub или архиве).
  - Файл с описанием: назначение, инструкция по запуску, структура классов.
  - Демонстрация работы (видео или личный показ).
  - Защита проекта: студент объясняет ключевые решения и отвечает на вопросы.

| Оценка | Критерии |

## 📋 Требования к решению задач на контрольных работах

При оценке практических заданий мы руководствуемся следующими критериями:

### 1. Корректность работы программы
- Программа должна выполнять поставленную задачу без ошибок для всех предусмотренных условием входных данных.
- Результат должен точно соответствовать спецификации (формат вывода, поведение в граничных случаях).

### 2. Проверка входных данных (типы и значения)
- **Тип данных**: функция/метод должны проверять, что аргументы имеют ожидаемый тип (или могут быть к нему приведены без потери смысла). При несоответствии — генерировать исключение `TypeError` с понятным сообщением.
- **Корректность значений**: проверка на допустимые диапазоны (например, часы от 0 до 23, возраст человека не более 200 лет, цена товара не отрицательная). При нарушении — выбрасывать `ValueError` (или собственное исключение).
- Пример:
  ```python
  def set_age(self, age: int):
      if not isinstance(age, int):
          raise TypeError("Возраст должен быть целым числом")
      if not (0 <= age <= 200):
          raise ValueError("Возраст должен быть от 0 до 200 лет")
      self._age = age
  ```

### 3. Обработка исключений
- В коде должны быть обработаны потенциально опасные места (работа с файлами, ввод пользователя, деление на ноль и т.п.) с использованием `try/except/else/finally`.
- Исключения не должны «проглатываться» без уведомления. Допускается либо проброс выше, либо вывод информативного сообщения.

### 4. Документирование кода
- Каждый класс и публичный метод должны содержать **docstring** в формате (рекомендуется [Google style](https://google.github.io/styleguide/pyguide.html#38-comments-and-docstrings) или [reStructuredText](https://www.python.org/dev/peps/pep-0287/)).
- В docstring описывается назначение, аргументы (`Args:`), возвращаемое значение (`Returns:`), возможные исключения (`Raises:`).
- Пример:
  ```python
  def calculate_discount(price: float, percent: float) -> float:
      """Возвращает цену со скидкой.

      Args:
          price: Исходная цена (должна быть >= 0).
          percent: Процент скидки (от 0 до 100).

      Returns:
          Цена после применения скидки.

      Raises:
          ValueError: Если price < 0 или percent не в [0, 100].
      """
  ```

### 5. Читаемость и стиль кода
- Соблюдение **PEP 8**: отступы, пробелы, именование переменных и функций (`snake_case`), классов (`CamelCase`), констант (`UPPER_CASE`).
- Код должен быть самодокументируемым: понятные имена переменных, отсутствие «магических чисел» (использовать константы).
- Комментарии поясняют сложные алгоритмы или неочевидные решения (но не пересказывают код).

### 6. Дополнительные требования (для высоких оценок)
- **Эффективность** – разумное использование алгоритмов и структур данных.
- **Использование современных возможностей Python** – аннотации типов (`typing`), `dataclasses`, перечисления (`Enum`) и т.д.
- **Лаконичность и избегание дублирования** – соблюдение принципа DRY (Don’t Repeat Yourself).

### 1. Установка Python

#### 🪟 Windows
1. Скачайте установщик с [официального сайта](https://python.org/downloads).
2. **ВАЖНО:** Поставьте галочку ✅ **"Add Python to PATH"** при установке.
3. Запустите командную строку и проверьте:
   ```bash
   python --version   # Python 3.11.0 или выше
   ```

#### 🍎 macOS
```bash
brew install python
python3 --version
```

### 2. Установка Git

#### 🪟 Windows
1. Скачайте Git: [https://git-scm.com/download/win](https://git-scm.com/download/win)
2. Установите с параметрами по умолчанию.
3. **После установки перезапустите терминал** (или Git Bash), чтобы изменения вступили в силу.
4. Проверьте:
   ```bash
   git --version
   ```
5. Настройте имя и email:
   ```bash
   git config --global user.name "Имя Фамилия"
   git config --global user.email "your_email@example.com"
   ```

#### 🍎 macOS
```bash
# через Homebrew
brew install git

# или через Xcode Command Line Tools
xcode-select --install
```
Проверка и настройка аналогично Windows.

### 3. Установка редактора кода (VSCode)
1. 📥 Скачайте [Visual Studio Code](https://code.visualstudio.com/).
2. 🔧 Установите расширения **Python** и **Jupyter** от Microsoft:
   - Откройте VSCode, перейдите во вкладку Extensions (Ctrl+Shift+X), найдите "Python" и "Jupyter", установите.
3. 🐍 Выберите интерпретатор:
   - Нажмите Ctrl+Shift+P, введите "Python: Select Interpreter", выберите версию Python, которую установили.

### 4. Клонирование репозитория курса
1. Откройте терминал / Git Bash.
2. Перейдите в папку для проектов (например, `cd Desktop`).
3. Склонируйте репозиторий (ссылку выдаст преподаватель):
   ```bash
   git clone <ССЫЛКА_НА_РЕПОЗИТОРИЙ>
   ```
   > 💡 *Плейсходер `<ССЫЛКА_НА_РЕПОЗИТОРИЙ>` будет заменён на актуальную ссылку в начале семестра.*
4. Перейдите в папку проекта:
   ```bash
   cd название_репозитория
   ```
5. Откройте в VSCode:
   ```bash
   code .
   ```

### 5. Проверка окружения
Создайте файл `test.py`:
```python
import sys
print(f"✅ Python версия: {sys.version}")
print("🎉 Окружение готово к работе!")
```
Запустите:
```bash
python test.py
```

### 6. Дополнительные инструменты (рекомендуется)
```bash
# Установка pip (обычно уже есть)
python -m ensurepip --upgrade

# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate

# Активация (macOS/Linux)
source venv/bin/activate

# Установка библиотек (пример)
pip install имя_пакета
```

> 💡 *После активации виртуального окружения в терминале появится префикс `(venv)`.*

## ⁉️ Частые вопросы

**Q: Какая версия Python нужна?**  
A: **Python 3.11 или новее**. Старые версии Python 2.x не поддерживаются.

**Q: Чем отличается этот курс от курса по C++?**  
A: Мы фокусируемся на особенностях Python: динамической типизации, декораторах, «магических» методах и питонических идиомах.

**Q: Нужно ли знать C++ для этого курса?**  
A: Да, курс рассчитан на студентов, уже изучивших основы программирования на C++.

**Q: Что делать, если я не успеваю выбрать путь оценивания?**  
A: Путь нужно выбрать до первой контрольной работы. Если этого не сделать, по умолчанию вы идёте путём контрольных работ.
